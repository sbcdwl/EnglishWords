# 艾宾浩斯遗忘曲线 & SM-2 算法

## 核心思想

艾宾浩斯遗忘曲线指出：人类记忆会随时间快速遗忘，但通过**及时复习**可以强化记忆。SM-2 算法是一种基于用户反馈的动态间隔重复算法，由 SuperMemo 公司提出，广泛应用于各类背单词和记忆软件中。

本软件在标准 SM-2 算法基础上，增加了**错误权重惩罚机制**，让错得多的单词更容易出现在复习中。

---

## 复习间隔计算

### 标准 SM-2 公式

```text
interval = previous_interval * ease_factor
```

其中：
- `interval`：当前间隔天数
- `previous_interval`：上次复习的间隔天数
- `ease_factor`：难度因子（初始 2.5，范围 1.3 ~ 2.5）

### 特殊规则

| 复习次数 | 条件 | 间隔天数 |
| :--- | :--- | :--- |
| 第 1 次 | quality >= 3（记住） | 1 天 |
| 第 2 次 | quality >= 3（记住） | 6 天 |
| 第 3 次及以上 | quality >= 3（记住） | `interval * ease_factor` |
| 任意次数 | quality < 3（忘记） | 1 天（重置） |



## 难度因子更新

### 当用户评分 quality >= 3（被记住）

```text
ease_factor = ease_factor + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02))
```
**评分对应的调整值**（近似）：

| quality | 调整值 | 说明 |
| :--- | :--- | :--- |
| 3（记住） | +0.04 | 正常 |
| 4（容易） | +0.06 | 较好 |
| 5（非常容易） | +0.10 | 最佳 |

### 当用户评分 quality < 3（被忘记）

本软件的**增强惩罚机制**：

```text
penalty = wrong_penalty_factor * (1 + wrong_count / 10)
ease_factor = ease_factor - penalty
```

其中：
- `wrong_penalty_factor`：用户可配置的惩罚系数（默认 0.15）
- `wrong_count`：该单词累计错误次数

**示例**：

| wrong_count | 惩罚值 | 新 ease_factor |
| :--- | :--- | :--- |
| 0 | 0.15 | 2.35 |
| 5 | 0.225 | 2.275 |
| 10 | 0.30 | 2.20 |
| 20 | 0.45 | 2.05 |

### 边界限制
```text
ease_factor = max(1.3, min(2.5, ease_factor))
```

## 评分标准（quality）

| 用户操作 | quality | 说明 |
| :--- | :--- | :--- |
| 忘记 | 0 | 完全不记得单词意思 |
| 困难 | 2 | 勉强想起，但不确定 |
| 记住 | 3 | 能正确回想 |
| 容易 | 4 | 轻松想起 |
| 非常容易 | 5 | 几乎不用思考 |

**注意**：quality 1 在标准 SM-2 中表示"几乎忘记"，本软件简化处理，将其归入"忘记"类别。

---

## 错误权重策略

### 核心公式

```text
priority = wrong_count / (correct_count + wrong_count + 1)
```

### 优先级排序

复习词列表按以下规则排序：
1. **主排序**：`priority` 降序（错误率越高越靠前）
2. **次排序**：`ease_factor` 升序（难度越高越靠前）

### 示例对比

| 单词 | correct | wrong | priority | ease_factor | 排序位置 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| talk | 2 | 5 | 0.625 | 2.0 | 第 1 位 |
| hello | 5 | 3 | 0.333 | 2.3 | 第 2 位 |
| world | 10 | 1 | 0.083 | 2.5 | 第 3 位 |

## 每日学习流程

```text
┌─────────────────────────────────────────────────────────────────┐
│                    每日启动时                                   │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│  1. 查询 next_review_date <= 今天 的单词                       │
│     → 得到「复习词列表」                                        │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 按 priority（错误率）降序排列复习词                       │
│     错误率高的 → 优先复习                                       │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 从 is_new = 1 的单词中取 daily_new_words 个               │
│     → 得到「新词列表」                                          │
│     按导入顺序（created_date）升序                             │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. 合并队列：复习词 + 新词                                    │
│     （先复习，再学新词）                                        │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│  5. 用户逐个学习，评分                                         │
│     → 更新 review_records 表                                   │
│     → 重新计算 next_review_date                                │
└─────────────────────────────────────────────────────────────────┘
```

## 完整示例：从新词到掌握

### 初始状态

```text
单词: talk
is_new: 1
review_count: 0
ease_factor: 2.5
interval_days: 0
next_review_date: (明天)
```

### 第 1 次学习（新词）
```text
用户操作: 记住（quality=3）
→ is_new = 0
→ review_count = 1
→ interval_days = 1
→ ease_factor = 2.5 + 0.04 = 2.54 → 上限 2.5
→ next_review_date = 今天 + 1 天
```

### 第 2 次复习

```text
用户操作: 容易（quality=4）
→ review_count = 2
→ interval_days = 6
→ ease_factor = 2.5 + 0.06 = 2.5（已达上限）
→ next_review_date = 今天 + 6 天
```

### 第 3 次复习（忘记）

```text
用户操作: 忘记（quality=0）
→ review_count = 3
→ wrong_count = 1
→ interval_days = 1（重置）
→ penalty = 0.15 * (1 + 1/10) = 0.165
→ ease_factor = 2.5 - 0.165 = 2.335
→ next_review_date = 今天 + 1 天
```

### 第 4 次复习

```text
用户操作: 容易（quality=4）
→ review_count = 4
→ correct_count = 2
→ interval_days = 1 * 2.335 = 2.335 → 约 2 天
→ ease_factor = 2.335 + 0.06 = 2.395
→ next_review_date = 今天 + 2 天
```

### 多次复习后

```text
经过多次复习，单词逐渐进入长期记忆：
review_count: 12
correct_count: 10
wrong_count: 2
ease_factor: 2.45
interval_days: 45
next_review_date: 45 天后
```

## 算法参数可配置性

所有算法参数均存储在 settings 表中，用户可通过设置对话框修改：

|参数|	默认值|	范围|	说明
|---|---|---|---
|ease_factor_min|	1.3|	1.0 ~ 3.0|	最小难度因子
|ease_factor_max|	2.5|	1.0 ~ 3.0|	最大难度因子
|wrong_penalty_factor|	0.15|	0.01 ~ 0.50|	错误惩罚系数

修改后立即生效，无需重启程序。
